## 什么是模板？

**模板是 AgentOS 的标准化代码库。** 它们包含：

- 一个使用 FastAPI 的 **AgentOS 实例**。
- 一个用于存储会话、记忆、评估和知识的 **PostgreSQL 数据库**。
- 一组作为起点的**预构建智能体、团队和工作流**。

您可以使用模板来开始并将您的 AgentOS 投入生产环境，而不是从头开始。它们被设置为使用 docker 在本地运行并在云提供商上运行。它们是一个极好的起点，被使用 Agno 的各种公司使用。您肯定需要根据您的特定需求定制它们，但它们会让您更快地开始。

Agno 目前支持以下模板：

Docker: [agent-infra-docker](https://github.com/agno-agi/agent-infra-docker)
AWS: [agent-infra-aws](https://github.com/agno-agi/agent-infra-aws)
Railway: [agent-infra-railway](https://github.com/agno-agi/agent-infra-railway)

即将推出：

- [Modal](https://modal.com/)
- [Render](https://render.com/)
- [Fly.io](https://fly.io/)
- [GCP](https://cloud.google.com/)

此外，所有模板都包含开发环境和生产环境。开发环境用于开发和测试。生产环境用于生产。对于大多数用例，我们建议在开发环境中使用 docker，并为生产环境选择您选择的云提供商。

## 如何开始使用模板：

1. 确保您已安装 agno 和 agno-infra。

<CodeGroup>

```bash Mac
pip install -U agno
pip install -U agno-infra
```

```bash Windows
pip install -U agno
pip install -U agno-infra
```

</CodeGroup>

2. 使用 `ag infra create` 创建您的代码库并选择一个模板。

<CodeGroup>

```bash Mac
ag infra create
```

```bash Windows
ag infra create
```

</CodeGroup>

3. `cd` 进入您的代码库并运行开发环境。

4. 在测试开发环境并准备好部署到生产环境后，运行生产环境。

注意：

- 开发环境和生产环境的说明对于每个模板都不同。请参考模板文档了解具体说明。
- 您可以克隆仓库并按照模板文档中的说明操作，而不是使用 `ag infra create`。
