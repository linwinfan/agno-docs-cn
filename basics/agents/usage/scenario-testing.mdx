---
title: 场景测试
---

此示例演示如何使用场景测试库来测试具有定义的成功和失败标准的智能体。它展示如何为智能体行为和响应实施自动化测试。

<Steps>

  <Step title="创建 Python 文件">
    ```bash
    touch scenario_testing.py
    ```
  </Step>


  <Step title="将以下代码添加到您的 Python 文件中">
    ```python scenario_testing.py
    """
    这是一个使用[scenario](https://github.com/langwatch/scenario)测试库来测试智能体的示例。

    先决条件：
    - 安装scenario：`uv add langwatch-scenario OR pip install langwatch-scenario`
    """

    import pytest
    from scenario import Scenario, TestingAgent, scenario_cache

    Scenario.configure(testing_agent=TestingAgent(model="openai/gpt-5-nano"))


    @pytest.mark.agent_test
    @pytest.mark.asyncio
    async def test_vegetarian_recipe_agent():
        agent = VegetarianRecipeAgent()

        def vegetarian_recipe_agent(message, context):
            # 在此处调用您的智能体
            return agent.run(message)

        # 定义场景
        scenario = Scenario(
            "用户正在寻找晚餐创意",
            agent=vegetarian_recipe_agent,
            success_criteria=[
                "食谱智能体生成素食食谱",
                "食谱包含食材清单",
                "食谱包含逐步烹饪说明",
            ],
            failure_criteria=[
                "食谱不是素食或包含肉类",
                "智能体询问超过两个后续问题",
            ],
        )

        # 运行场景并获取结果
        result = await scenario.run()

        # 断言以让pytest知道测试是否通过
        assert result.success


    # 示例智能体实现
    from agno.agent import Agent  # noqa: E402
    from agno.models.openai import OpenAIChat  # noqa: E402


    class VegetarianRecipeAgent:
        def __init__(self):
            self.history = []

        @scenario_cache()
        def run(self, message: str):
            self.history.append({"role": "user", "content": message})

            agent = Agent(
                model=OpenAIChat(id="gpt-5-mini"),
                markdown=True,
                instructions="您是一个素食食谱智能体",
            )

            response = agent.run(message)
            result = response.content
            print(result)
            self.history.append(result)

            return {"message": result}
    ```
  </Step>

    </CodeGroup>
